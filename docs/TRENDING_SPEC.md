# 트렌딩(급상승) 탐지 명세서

이 문서는 메뉴의 "꾸준한 인기"와 "급상승 바이럴"을 분리해 탐지하기 위한 고정 기준 문서입니다.

---

## 1. 목표
- 누적 인기만 보는 방식의 한계를 보완
- 신메뉴/바이럴 메뉴를 빠르게 포착
- 광고성 급증을 방어하며 신뢰도 유지

---

## 2. 지표 정의

### 2-0. 공통 필터(강제)
- 광고 리뷰(`is_ad=true`)는 인기/트렌딩 모든 집계에서 100% 제외
- 집계 대상은 비광고 리뷰만 사용

### 2-1. Popularity Score (안정 인기)
- 최근 30일 비광고 언급량 기반
- 기본식(예시):
  - `popularity_score = log(1 + non_ad_mentions_30d)`

### 2-2. Trend Score (급상승)
- 최근 1~3일 증가 폭과 7일 기준선을 비교
- 기본식(예시):
  - `z = (today_mentions - mean_7d) / (std_7d + 1)`
  - `growth_3d = (mentions_3d / (mentions_prev_3d + 1))`
  - `trend_score = 0.7 * z + 0.3 * growth_3d`

### 2-3. 신뢰도 가중치
- 소스 다양성(플랫폼/작성자 분산) 가중치 반영
- 광고 의심 비율이 높으면 점수 감점

---

## 3. 이상 탐지/보정 규칙
1. 단일 소스에서 갑자기 폭증한 경우 이상치 플래그 부여
2. 광고 비율(`ad_ratio`)이 임계치 초과 시 trend_score 감점
3. 데이터 부족(표본 너무 적음) 시 "저신뢰" 라벨 표시

---

## 4. 신메뉴 감지
1. 메뉴 사전에 없는 명칭 반복 등장 시 `menu_candidate`에 적재
2. 등장 빈도/기간/소스 다양성 기준 충족 시 검수 대상 승격
3. 검수 완료 후 `menu`/`menu_alias`에 반영

---

## 5. 응답 정책
- `mode=popular`: 안정 인기 순위 반환
- `mode=trending`: 급상승 순위 반환
- 추천 사유에 수치 근거 포함
  - 예: "최근 3일 언급량이 직전 3일 대비 2.1배 증가"
