# 문서 우선 개발 규약 (고정)

## 1. 기준 문서
- 구조/진행: `docs/PROJECT_STRUCTURE.md`
- DB 기준: `docs/DB_SPEC.md`
- API 기준: `docs/API_SPEC.md`

## 2. 변경 원칙
1. 변경이 필요하면 **문서 먼저 수정**
2. 문서 승인 후 코드 반영
3. 코드 반영 후 테스트/검증
4. 결과를 README와 문서 상태에 동기화

## 3. 금지 사항
- 문서 미반영 상태에서 API 응답 구조 임의 변경 금지
- DB 스키마 하드코딩/무계획 변경 금지
- 민감정보 코드 내 작성 금지

## 4. 품질 게이트
- 테스트 통과
- 광고 필터링 검증
- 메뉴 추출 검증
- 문서/코드 정합성 확인
