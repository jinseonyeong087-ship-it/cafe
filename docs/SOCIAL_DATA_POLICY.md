# SNS 데이터 수집/활용 정책 (X, Instagram)

이 문서는 SNS 기반 데이터 확장 시 준수해야 할 정책 기준 문서입니다.

---

## 1. 결론 (프로젝트 전략)
- **MVP 단계:** 블로그/리뷰 기반으로 먼저 구축
- **확장 단계:** X/Instagram은 "공식 API 또는 허용 범위" 내에서만 추가
- 즉, "지금 당장 제외"가 아니라 **2단계 포함(조건부 포함)** 전략을 채택

---

## 2. 왜 이렇게 가는가?
1. SNS는 실제 바이럴 포착에 매우 유리함
2. 그러나 무단 크롤링은 정책/법적/기술 리스크가 큼
3. 초기 완성도 확보 후, 안정적으로 확장하는 것이 전체 성공 확률이 높음

---

## 3. 허용 원칙
1. 플랫폼 ToS/정책 준수
2. 공식 API 우선
3. 개인정보 최소수집(원문 장기보관 최소화)
4. 요청량 제한 및 차단 대응 정책 보유
5. 소스/수집시점 메타데이터 기록

---

## 4. 금지 원칙
- 로그인 우회/보안 우회 자동화 금지
- 차단 회피 목적의 비정상 접근 금지
- 민감정보 수집/저장 금지

---

## 5. 제품 반영 방식
- SNS 데이터는 초기에는 "보조 신호"로 사용
- 최종 추천 점수 = 리뷰 기반 점수 + SNS 트렌드 가중치
- SNS 신호만으로 단독 추천하지 않음 (초기 기준)

---

## 6. 단계별 도입 계획
### Phase A (현재)
- 리뷰 기반 파이프라인 완성
- 광고 필터/메뉴 추출/추천 API 안정화

### Phase B
- X API 연동(가능 범위)
- 해시태그/키워드 언급량 집계

### Phase C
- Instagram 허용 범위 연동
- 트렌딩 점수 통합 및 A/B 검증
